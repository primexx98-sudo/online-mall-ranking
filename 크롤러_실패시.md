## Claude가 할 일 (순서대로)

1. **날짜 확인**: 캡쳐가 오늘 날짜 기준인지 사용자에게 확인(다른 언급 없으면 오늘).
2. **`data/daily/YYYY-MM/YYYY-MM-DD.xlsx` 존재 확인**. 없으면 그날 카카오·다이소
   자동 수집 자체가 안 됐거나 그것도 실패한 것 — `git pull`로 최신 상태부터 받고,
   그래도 없으면 `python main.py`로 먼저 그날 수집을 실행해야 함.
3. **캡쳐 이미지를 Read 도구로 읽어 TOP10 추출**: 순위, 브랜드, 상품명, 가격(할인가
   있으면 할인가, 없으면 정가). 상품URL은 캡쳐에 안 보이므로 생략(빈 값).
   - 10개 미만만 보이면(스크롤 잘림 등) 사용자에게 나머지 캡쳐를 추가 요청.
4. **items.json 작성**: 스크래치패드에 아래 형식으로 저장.
   ```json
   [
     {"순위": 1, "브랜드": "...", "상품명": "...", "가격": "8500"},
     ...
   ]
   ```
5. **주입 스크립트 실행** (프로젝트 루트에서):
   ```bash
   python manual_oliveyoung.py <YYYY-MM-DD> <items.json 경로>
   ```
   → `data/daily/YYYY-MM/YYYY-MM-DD.xlsx`의 "올리브영" 시트가 교체되고
   "카테고리통계" 시트가 재계산됨.
6. **결과 검증**: 해당 xlsx를 pandas로 열어 "올리브영" 시트 행 수(=캡쳐로 준 개수)와
   "카테고리통계" 합계가 맞는지 확인.
7. **git 반영**: `git pull --rebase` → `git add data/daily/...` →
   `git commit -m "..."` → `git push`.
8. **사용자에게 결과 요약 보고**: 반영된 개수, 카테고리 분류 결과, push 완료 여부.
