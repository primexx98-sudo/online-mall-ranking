## 1. 프로젝트 개요

올리브영 · 다이소몰 · 카카오 선물하기 건강기능식품 판매순위 TOP10을 매일 자동
수집하여 일별 엑셀로 저장하고, 월별 평균순위 TOP20을 집계한다. 상품을
카테고리로 자동 분류하여 일별·월별 카테고리 비율 통계 시트를 함께 생성한다.

**이전 프로젝트에서 이어짐**: `Desktop\업무\시장조사`(Health-Ranking 리포)에서
동일한 목적으로 운영하던 프로젝트가 올리브영 크롤링이 GitHub Actions에서
반복적으로 Cloudflare에 막히면서, 리포를 완전히 새로 만들어 코드를 전면
재작성하기로 하고 시작한 새 프로젝트다. 이전 리포는 그대로 두고 병행하지 않는다.

## 6. 크롤러 방식

카카오·다이소는 Playwright(headless Chromium) + `playwright-stealth`로 동일한
패턴을 사용한다(`crawlers/base.py`의 `new_page()`). 올리브영은 Selenium을
별도로 사용한다(아래 참고).

| 플랫폼 | 핵심 셀렉터 | 비고 |
|--------|-----------|------|
| 카카오 | `gc-product` 컨테이너, `span.txt_prdbrand`, `strong.txt_prdname`, `.txt_price`, `a.link_prdunit[href]` (`.area_ad` 제외) | 카테고리당 10개×2=최대 20개 |
| 다이소 | `.nav-rank .swiper-slide` (DOM 순서 = 실시간 순위 순서), `.product-title`, `.price-value`, `a.detail-link[href]` | 브랜드는 목록에 없어 TOP10 상세페이지(`.brand-area .detail-title`) 추가 방문 |
| 올리브영 | `ul.cate_prd_list > li`, `span.tx_brand`, `p.tx_name`, `.tx_cur`(현재가), `a[href]` | Selenium, 3장 URL 참고 |

**올리브영 차단 대응 — Selenium으로 전환 (2026-07-22 결정, 해결됨)**: 이전
프로젝트에서 GitHub Actions(Azure IP)가 Playwright 기반 수집을 Cloudflare
Turnstile로 막는 문제를 코드 레벨로 해결하지 못해 캡쳐 기반 수동 입력으로
전환했던 이력이 있다(기록.md 참고). 이번 리포도 초기엔 동일하게 Playwright로
재현 시도했으나 같은 패턴으로 재차단됨을 확인. 이후 **Selenium +
`time.sleep()` 방식**(`crawlers/oliveyoung_selenium_test.py`로 실증, 이후
`crawlers/oliveyoung.py`에 이식)을 GitHub Actions에서 4회 연속 테스트한 결과
전부 Turnstile 없이 정상 수집(상품 100건)됨을 확인 — Playwright 자체가 아니라
**Playwright의 자동화 시그니처**가 차단 트리거였던 것으로 추정. 현재는 이
Selenium 버전이 운영 코드이며, `manual_oliveyoung.py` + `크롤러_실패시.md`
절차는 예비책(재차단 시 대응)으로 남겨둔다.

## 8. 자동화 (GitHub Actions)

| 워크플로 | 트리거 | 동작 |
|---------|--------|------|
| `daily_crawl.yml` | 매일 UTC 00:00(=KST 09:00) + 수동 | 3개 플랫폼 수집 → 카테고리 분류 → xlsx 저장 → git push |
| `monthly_aggregate.yml` | 매월 1일 UTC 00:00 + 수동 | 전월 daily xlsx 집계 → 카테고리 통계 → 월별 xlsx 저장 → git push |

- **실행 환경**: `ubuntu-22.04` 고정 (이전 프로젝트에서 `ubuntu-latest`=24.04가
  Playwright `libasound2` 호환 문제를 일으켰던 이력 반영)
- **필수 설정**: 두 워크플로 모두 `permissions: contents: write` + commit 후
  `git pull --rebase` 포함 (매월 1일 두 워크플로 동시 실행 시 push 충돌 방지)
- **실패 시**: `main.py`가 플랫폼 1개라도 실패하면 `sys.exit(1)` → GitHub
  기본 워크플로 실패 이메일 알림(계정 이메일)으로 확인. 카카오톡 알림 등
  추가 알림 채널은 이번 버전에는 없음(필요 시 추후 추가, 13장 참고)

## 9. 엑셀 파일 구조

### 일별 `data/daily/YYYY-MM/YYYY-MM-DD.xlsx` — 시트 4개

| 시트명 | 내용 |
|--------|------|
| 카카오선물하기 | 건강식품·영양제(1~10위) + 다이어트·이너뷰티(1~10위) = 최대 20행. 컬럼에 이미지URL 포함 (2026-07-22~) |
| 다이소몰 / 올리브영 | TOP10. 컬럼에 이미지URL 포함 (2026-07-22~) |
| 카테고리통계 | 3개 플랫폼 합산 카테고리별 상품수 + 비율(%), 플랫폼별 내역 |

### 월별 `data/monthly/YYYY-MM_월별취합.xlsx` — 시트 4개

| 시트명 | 내용 |
|--------|------|
| 카카오선물하기 / 다이소몰 / 올리브영 | 월평균순위 TOP20 (전월 대비 순위·가격 변동 포함) |
| 카테고리통계 | 월간 TOP20 기준 카테고리 비율 (전월 대비 증감 포함) |

플랫폼 시트 컬럼: `순위(월평균) | 카테고리 | 상품명 | 브랜드 | 평균가격 | 평균순위 | 등장횟수 | 전월가격 | 가격변동 | 전월순위 | 순위변동`

카테고리통계 컬럼: `카테고리 | 전체 | 비율(%) | 카카오선물하기 | 다이소몰 | 올리브영 | 전월비율(%) | 증감(%p)`

### 월별 집계 로직 (`monthly_aggregate.py`)

```
최소등장   = min(3, 실제수집일수)
           → 필터 후 대상이 20개 미만이면 최소등장을 2회, 1회로 순차 완화해 재필터링
일별점수   = (11 - 순위) × 0.5              # 1위=5점 … 10위=0.5점
평균점수   = SUM(일별점수) / 등장횟수
등장률     = 등장횟수 / 실제수집일수          # 달력 일수가 아닌 실제 파일 수 기준
월간점수   = 평균점수×0.7 + (등장률×5)×0.3   # 순위 70% + 꾸준함 30%
월간TOP20  = (등장횟수 ≥ 최소등장, 완화 적용 후) 대상 중 월간점수 기준 상위 20개 (표시는 평균순위)
```

이 점수 공식·최소등장 필터·전월 대비 비교 로직은 이전 프로젝트(기록.md
2026-07-01, 2026-07-13 참고)에서 여러 차례 시행착오를 거쳐 확정된 것을
그대로 재사용했다 — 문제가 검증된 로직이라 다시 처음부터 튜닝할 이유가 없음.

**이번 버전에서 이식하지 않은 것**: 이전 프로젝트의 월별요약 시트(핵심지표·
1위상품·카테고리추천·급상승/급하락·신제품 인사이트 9블록), 브랜드통계 시트,
Streamlit 대시보드, 카카오톡 실패 알림은 이번 재구축 범위에 포함하지 않았다.
필요해지면 추후 요청 시 추가(13장 참고).
